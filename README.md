# MySprout（マイスプラウト）-気分記録アプリ-

## サービス概要
うつ病や双極性障害の方がその日の気分を記録するアプリ。<br>
記録を続けられるよう、記録のハードルを低くくなるようにしております。<br>
また、生活や治療に役立てられる機能もあります。(記録のグラフ化、医師に伝えたいことのメモ機能)<br>
　※メモ機能はMVPリリース後のアップデートで実装予定

## このアプリへの思い・作りたい理由
### 作成理由
以下の2点からこのアプリの作成を考えました。<br>
- 自身の今の断続的な記録を継続できるようにしたい
- 自身と同じような経験をしている方の助けになりたい。(自身もつらかったので同じような経験をする方の力になりたい)

私自身が挫折と自己嫌悪を繰り返した経験から生まれたこのアプリが、同じ悩みを抱えている方の助けになることを心から願っています。<br>
「記録が続けられる自分」を実感できる体験を、一人でも多くの方に届けたいと思っています。

以下からは、このアプリの作成経緯と実現したいことを記載しております。

### 記録を始めたきっかけ
双極性障害と診断され、医師に気分や睡眠時間を聞かれることから正確に伝えるために記録を行うことを決めました。<br>
また、以下の効果も期待して評判の良い記録アプリをいくつか試してみました。
- 医師に正確な状態を伝えられ、薬の調整や治療方針の精度が向上する
- 躁状態・うつ状態の前兆をいち早く察知し、本格的な症状を防げる<br>
  （強い躁状態の後には強いうつ状態が来るため、早めに察知して睡眠時間を増やすことで症状を和らげられる）
- 自分の状態を可視化することで、調子が悪い時期でも「少しずつ良くなっている」と実感できる

### 既存アプリでの挫折
しかし、どのアプリも3日～2週間で挫折してしまいました。<br>
現在は、断続的に記録の再開と中断を繰り返しています。<br>
**挫折した理由：**
1. **毎日アプリを開くのが億劫**
   - 夕方にはエネルギーが尽きて、アプリを開く気力すらない
   - 「今日も記録できなかった」と思うと、翌日はさらに開きにくくなる
2. **記録項目が多すぎて疲れる**
   - 「今日の出来事」「感情」「運動」など、気力がない時には答えられない質問ばかり
   - 気分だけの記録も可能だったが、 **「項目があるなら全部埋めるべきだ」という自分の完璧主義が許さなかった**
3. **11段階評価は細かすぎて判断に時間がかかる**
   - 「6と7の違いは何だろう？」と考えているうちに疲れてしまう
   - 今日の行動を深く思い起こさないと記録できず、それ自体が負担になった<br>

**最もつらかったこと：**<br>
&nbsp; &nbsp; &nbsp; &nbsp;記録が続かないたびに、 **「記録すらできない自分はダメな患者だ」** という自己嫌悪に陥り、症状がさらに悪化していきました。

### 挫折から得た気づき
回復期に気づいたのは、 **「記録したくないのではなく、記録のハードルが高すぎた」** ということです。<br>
本当に必要だったのは、以下の2点でした。
#### 1. 記録までのステップを極限まで減らす<br>
回復期は1日のエネルギー量が少なく、夕方には記録する気力すら残っていません。<br>
そのため：
- アプリを開いてから記録完了までのステップを最小限にする
- 最終的には **アプリを開かずに記録できる** 状態を目指す

これらによる、 **「記録しよう」と思った瞬間に終わる** 体験が必要でした。
#### 2. 直感的に記録できる粒度にする<br>
医師が本当に知りたいのは：
- 気分の波のパターン
- 治療の効果が出ているか

という **長期的な傾向** です。<br>
そのため、細かい記録よりも **「大雑把でも続けられる仕組み」** が重要だと考えました。<br>
つまり、 **「最高・良い・普通・悪い・最悪」の5段階** さえ記録できれば十分という考えです。

### このアプリで実現したいこと
このアプリでは、以下の2つを目的としています。
#### 1. 医師とのコミュニケーション改善
**私自身の体験:**<br>
記録していない場合は、診察で「最近どうですか?」と聞かれても、 **「なんとなく調子が悪い」** としか答えられませんでした。<br>
記録できた際は、 **「先週の火曜日から気分が落ち込み始めて、金曜日にピークでした」** と具体的に伝えることができました。<br>
**これにより:**
- より具体的なアドバイスを貰えるようになった。
- 医師に正確な情報を伝えることができて自信がついた。
- 薬の調整などの話が出るようになった。
#### 2. 記録が続かないことによる自己嫌悪の防止
**私自身の体験:**<br>
以前は、記録アプリを使おうとしても、 **「今日も記録できなかった」と自分を責めてしまう** ことが多くありました。(現在はそういったことも仕方ないで受け流せるようになりました)<br>
特に、調子が悪い時ほど記録する気力がなく、 **「記録できない自分はダメだ」** という気持ちになっていました。<br>
このアプリでは、 **記録のハードルを極限まで下げる** ことで、「続けられる自分」を実感できるようにしたいと考えています。<br>
**これにより:**
- 「記録できない自分はダメだ」という自己嫌悪から解放される
- 小さな成功体験を積み重ねられる
- 記録を続けることで、自分の状態を客観視できるようになる

## ユーザーについて
### メインターゲット
うつ病・双極性障害で通院中の方のうち、「気分の記録をしたいが、既存アプリが面倒で続かない」方

**想定する状態:**
- 通院や日常生活は送れているが、気力に波がある
- 調子が良い時は記録する気力があるが、調子が悪い時は続かない

**特に以下のような経験がある方:**
- 記録アプリを試したが3日坊主で終わった
- 記録項目が多すぎて入力が負担になった
- 記録が続かない自分を責めて、アプリを開くのも辛くなった

## 使い方のイメージ
### 記録
- 初めに<br>
記録方法は、5段階評価のアイコンをその日の気分で選択する方式です。<br>
以降は、アイコンと呼称させていただきます。
- アプリからの記録(当日の記録)
    1. アプリを開き、ホーム画面を表示
    2. ホーム画面に表示されているアイコンのいずれかをタップ
    3. GOODが出たら記録完了
- アプリからの記録(当日以外)
    1. アプリを開く
    2. ホーム画面の+ボタンを選択
    3. 記録画面が開くので今日の気分を選択
    4. 保存を押すと記録が完了
- ウィジェットからの記録(MVPでは未実装)
    1. ウィジェットからアイコンのいずれかをタップ
    2. GOODが表示され、気分の記録が完了

### 生活や治療への活用
- これまでの記録を曲線でグラフとして表示し、セルフケアや治療に活用可能

## サービスの推しポイント
### 他サービスとの比較

| 評価項目 | 類似サービスA | 類似サービスB | 本サービス |
| ---------- | ---------- | ---------- | ---------- |
| 記録方法 | 5段階評価 | 11段階評価 | 5段階評価 |
| 記録完了までのタップ数 | 3回 | 4回 | 2回 |
| 気分以外の記録項目 | 多数(出来事・感情など) | 活動量 | なし(MVPでは気分のみ) |
| 特徴 | アイコンのカスタマイズ | 細かい気分の記録が可能 | 記録のハードルが最も低い |

※記録完了までのタップ数は、気分のみを記録した場合の最短操作回数です。

### 既存アプリで感じた課題

私自身が実際に使って挫折した経験から、以下の課題を感じました。

#### 類似サービスA：記録項目が多すぎる
- 気分だけでなく、出来事・感情・睡眠など **多数の項目** を記録する必要がある
- 気分だけの記録も可能だったが、 **「項目があるなら全部埋めるべき」という自分の完璧主義が許さなかった**
- 夕方には気力が尽きて、アプリを開く気力すら残っていない

#### 類似サービスB：11段階評価は細かすぎる
- 「6と7の違いは何だろう？」と考えているうちに疲れてしまう
- 記録に迷っているうちに、 **アプリを閉じてしまう** ことが何度もあった
- 細かく記録できることが、かえって負担になっていた

### サービスの推しポイント
MySproutは、 **「記録のハードルを極限まで下げる」** ことに特化して設計しました。
#### 1. 記録の手間を極限まで削減
**たった2ステップ** で「記録しよう」と思った瞬間に終わるよう設計しました。<br>
**具体的な操作:**
1. アプリを開く
2. 今日の気分のアイコンをタップ → 自動保存

#### 2. 記録内容がシンプル
**5段階評価は「最高・良い・普通・悪い・最悪」で直感的。**
- 「6と7の違いは？」と悩む必要がない
- 少しの気力で記録可能
- 情報量が少ないので、 **脳への負担が最小限**<br>

夕方に気力が残っていなくても、 **「これなら記録できる」** と思える粒度にしました。

#### 3. メモ機能(アップデート後)
気分の記録だけでなく、 **「医師に伝えたいこと」** も記録できます。<br>
**例えば:**
- 「今日は久しぶりに外出できた」
- 「薬を飲み忘れた日があった」
- 「夜中に目が覚めることが増えた」

通院前に見返すだけで、 **診察時に伝え忘れがなくなります。**<br>
気分記録とメモが1つのアプリで完結するので、複数のアプリを使い分ける必要がありません。

### 実体験から生まれた設計
私自身が既存アプリで挫折した経験から、 **「記録が続かない理由」** を排除しました。
| 既存アプリの課題 | MySproutでの解決策 |
|---------------|------------------|
| 記録項目が多すぎる | 3項目に絞った(MVPでは1項目) |
| 記録までのステップが多い | 2ステップに減らした |
| 判断が難しい(11段階など) | 5段階で直感的に |

**「続けられる自分」を実感できる** ことが、このアプリの最大の価値です。

## このアプリの懸念点とその対策
### 懸念点①
- 懸念点：技術的にウィジェットからの記録が可能なのか
- 理由：できるのならば他のアプリで類似サービスで行っていそうだから。
- 対策：通知から記録ができないかを模索する。
### 懸念点②
- 懸念点：ターゲットに使ってもらえるかが不安<br>
　　　　(リリースして改善の流れに持っていくことができるのか不安)
- 理由1：何の実績もないアプリを使ってもらえるのか
- 対策：自身のかかりつけの医師や医師のコミニティで意見を貰い改善に入っていく。<br>
　　　(専門的知見を貰うこちらの方が見てもらえるかわからないので難易度的に難しいか)
- 理由2：自分としては記録項目が多いと疲れてしまうが、実際は記録項目が多い方がいいのか。
- 対策：誰かに使用してもらってそういった声が多かったら、記録項目を追加していく。<br>
　　　または、別アプリとして記録項目追加版を作成する。<br>
　　　上記は時間があった場合の対処法で、時間がない場合の対処法は考えつかなかった。

## MVPリリース後の検証
### MVPで検証したいこと
このアプリの本質は **「記録すること」ではなく、「記録を続けられること」** です。<br>
MVPでは、以下の2点を検証します:
1. 自身が抱えた課題と同じ課題を抱えた方がいるのかの確認
2. 課題の解決方法が合っているかの確認

### 検証の観点と判断基準
| 検証項目 | データ収集方法 | 成功の基準 |
|---------|--------------|----------|
| **記録の負担** | アプリ内フィードバックフォーム | 「これなら続けられる」という声が **7割以上** |
| **継続率** | 記録日数・最終記録日の分析 | **1ヶ月後の継続率が50%以上** |
| **診察時の役立ち度** | アプリ内フィードバックフォーム | 「診察時に役立った」という声が **7割以上** |

### データ収集の具体的な方法
1. 定量データ（自動収集）
    - ユーザーの記録日数
    - 最終記録日
    - 継続日数の分布

2. 定性データ（フィードバックフォーム）
フィードバックフォームで以下の質問を用意します:
    - 「記録の負担はどうでしたか？」(5段階評価 + 自由記述)
    - 「このアプリは続けやすいですか？」(5段階評価 + 自由記述)
    - 「診察時に役立ちましたか？」(5段階評価 + 自由記述)
    - 「改善してほしい点があれば教えてください」(自由記述)

### 検証結果に基づく次のアクション
#### パターン1：検証が成功した場合(すべての基準を達成)
- ウィジェットからの記録機能を開発(React Native化)
- 記録のハードルをさらに下げる方向で進める
#### パターン2：一部の基準のみ達成した場合

| 達成した基準 | 次のアクション |
|------------|--------------|
| 継続率は高いが、診察時の役立ち度が低い | メモ機能の改善、通院リマインダー機能の追加を優先 |
| 診察時には役立つが、継続率が低い | 記録のハードルをさらに下げる施策を検討(通知機能、ウィジェットなど) |
| 記録の負担は低いが、継続率が低い | 記録を続けるモチベーション機能を追加(記録日数の表示、励ましメッセージなど) |

#### パターン3：検証が失敗した場合(すべての基準を未達成)
- フィードバックを詳細に分析し、課題の本質を再定義
- 以下のような方向性を検討
  - 記録方法の変更(5段階評価以外の方法)
  - ターゲットユーザーの見直し(別の疾患や症状に特化)
  - 機能の追加(気分以外の記録項目を増やすなど)

## 最終構想について
### なぜReact Nativeに移行するのか
MVPで **「記録のハードルを下げることが有効」** だと検証できた場合、さらにハードルを下げるために **ウィジェットからの記録** を実現します。

**ウィジェット機能のメリット:**
- アプリを開かずに、ホーム画面から直接記録できる
- 記録までのステップが **2ステップ → 1ステップ** に削減
- 「アプリを開く」という心理的ハードルがなくなる

**技術選択の理由:**
- Hotwire(Turbo + Stimulus)では、ウィジェット機能を実装できない
- React Native(Expo)なら、iOS/Androidのウィジェットに対応可能
- MVPで構築したバックエンド(Rails)は、APIモードに作り直すことで流用可能

### 段階的な移行計画

React Nativeへの移行は、一度にすべてを作り直すのではなく、 **段階的に進めます。**

1. **Phase 1：バックエンドのAPI化**
   - RailsをAPIモードに作り直し
   - MVPのデータベースはそのまま流用
   - 既存のWebアプリも並行して動作させる

2. **Phase 2：モバイルアプリの開発**
   - React Native(Expo)でモバイルアプリを開発
